

# Plan: Turn the site into a full event-management CRM

The data backbone is already in place — 830 organizations, 583 contacts, 830 participation records, plus `activities` and `documents` tables. The admin UI, however, still reads the legacy `sponsors` / `vendors` tables (now wiped) and there are no workflow, messaging, or AI integrations yet. This plan reshapes Admin into a real CRM and layers on Slack, SMS, and ElevenLabs.

## Phase 1 — Rewire Admin around the new schema (foundation)

Replace the legacy admin pages with views driven by `organizations` + `participation` + `contacts` + `activities`.

- **Admin Overview**: pipeline funnel by status (Prospect → Contacted → In Discussion → Form Received → Committed → Paid → Not Participating), broken out by role (Sponsor / Vendor / Distributor / Class Table). Totals: $ committed, $ paid, # forms outstanding, # tents/electric requested, volunteers needed.
- **Organizations** (new page, replaces Sponsors/Vendors split): searchable/filterable table — filter by role, status, owner, type, year. Bulk select for actions (assign owner, change status, send email/SMS, log activity).
- **Org Detail drawer/page**: contact list, full activity timeline, participation history per year, documents, notes, quick actions (log call, send SMS, send email, generate contract).
- **Contacts** page: searchable table with org link, primary flag, last-touched date.
- **Activities** page: global timeline with filters (type, owner, org, date range).
- **Documents** page: contracts/W9s/invoices per org with status + signed date.
- Keep existing Sponsorship Levels, Tickets, Guests, Admins pages.

## Phase 2 — CRM workflows (status-driven automation)

A new `workflows` + `workflow_runs` schema and an Inngest-powered engine.

- **Triggers**: status change on participation, form received, no-touch in N days, T-minus event-day countdown, document signed, payment received.
- **Actions**: send email (Resend), send SMS (Twilio), post to Slack channel/DM, assign owner, change status, create follow-up task, log activity.
- **Templates**: reusable email/SMS/Slack templates with `{{org.name}}`, `{{contact.first_name}}`, `{{owner_name}}` merge fields.
- **Tasks**: lightweight `tasks` table for human follow-ups generated by workflows ("Liz: call back Crumbl Cookies — no response in 7 days").
- **Admin → Workflows page**: list/edit/enable workflows, view recent runs, retry failures.

## Phase 3 — Public-facing forms feeding the CRM

Keep current Vendor / Sponsor / Donate forms but:

- Submissions upsert into `organizations` + `contacts` + create a `participation` row at status "Form Received" for the current year.
- Trigger workflow: Slack notification to #extravaganza, confirmation email to submitter, task assigned to the listed owner.
- Add **Volunteer signup form** (new public page + `volunteers` table) feeding the same workflow engine.

## Phase 4 — Slack integration

Connect the Slack connector (bot mode), then:

- Channel notifier: new submissions, payments, and status changes post to a configurable channel.
- Owner DMs: when a task is assigned to an owner mapped to a Slack user, DM them with org link.
- Slash-style replies via slack message buttons aren't available with the connector — instead, action links deep-link back into the CRM.
- Admin → Settings page to map owner names (Liz, Steve, Jenn, etc.) to Slack user IDs and pick the default channel.

## Phase 5 — SMS (Twilio)

Connect the Twilio connector, then:

- Outbound SMS from the org detail page (with templates).
- Workflow action: send SMS from a campaign (e.g., "Reminder: vendor form due Friday").
- Inbound SMS handling via a public edge function webhook → logs an activity on the matching contact (matched by phone number) and notifies the owner in Slack.
- Compliance: opt-out handling (STOP/HELP), rate limit, and SMS Geo Permissions reminder for the admin.

## Phase 6 — ElevenLabs voice + chat

Two surfaces:

- **Public voice/chat agent** on the site (already in the original spec) — answers FAQ from a knowledge base built from event details.
- **Admin outreach voice agent**: from an org detail, click "Call with AI agent" → agent calls the contact (via Twilio + ElevenLabs realtime), uses a prompt tailored to the org's status (donor solicitation, volunteer recruit, vendor onboarding), logs transcript as an activity, and updates status if the contact commits.
- STT for voicemail / inbound call recordings → transcript stored as activity.

## Phase 7 — Polish

- Saved views, CSV export per page, audit log of admin actions, role granularity (admin vs viewer vs owner-only-their-orgs).
- Email/SMS analytics (sent, delivered, replied) on each org timeline.

## Technical notes

- **Schema additions**: `workflows`, `workflow_runs`, `tasks`, `volunteers`, `messages` (unified outbound/inbound for email/SMS/Slack with `channel`, `direction`, `status`, `external_id`), `templates`, `owner_slack_map`. All RLS: public read off, authenticated admins via `has_role`.
- **Engine**: Inngest connector for durable workflow execution + scheduled "no-touch in N days" jobs. Stripe webhooks → events → workflows.
- **Email**: Resend (already configured).
- **SMS**: Twilio connector via gateway + a public `twilio-inbound` edge function for webhooks.
- **Slack**: Slack connector (bot) via gateway; `slack-events` edge function only if Phase 4 needs threaded replies later.
- **Voice/chat**: ElevenLabs widget on public site; admin uses ElevenLabs Conversational AI via `useConversation` + Twilio outbound dialing for phone calls.
- **Secrets needed**: none new for Resend/Stripe (set). Will request Twilio + Slack + ElevenLabs + Inngest connector links when those phases start.
- **Migrations**: clean up — drop the now-unused `vendors` and `sponsors` tables (they were truncated last turn) once the new admin UI is shipped, to prevent confusion.

## Recommended build order

1. Phase 1 (Admin rewrite) — highest immediate value, unblocks daily use.
2. Phase 3 (Public forms feed CRM).
3. Phase 4 (Slack notifications) — fastest external integration win.
4. Phase 2 (Workflow engine) — needs Inngest.
5. Phase 5 (SMS).
6. Phase 6 (ElevenLabs voice agent).
